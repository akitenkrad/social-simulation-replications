## Overview

論文の再現実装．Cargo workspace + uv workspace の 2 プロジェクト構成で，Rust (`simulation/`) でシミュレーションを実行し，Python (`tools/`) で可視化する．

## Build & Run

全コマンドは workspace ルート（本ディレクトリ）から実行する．

```bash
# ビルド
cargo build --release

# シミュレーション実行
cargo run --release -- run [args...]

# パラメータスイープ
cargo run --release -- sweep [args...]

# 可視化
uv sync
uv run {{NAME}}-tools visualize [args...]
```

## Testing & Linting

```bash
cargo test
cargo clippy
cargo fmt --check
```

## Architecture

ワークスペース構成:

- `simulation/` — Rust クレート `{{NAME}}-simulation`．バイナリ名 `{{NAME}}`．
- `tools/` — Python パッケージ `{{NAME}}-tools`．src レイアウト，`tools/src/{{NAME}}_tools/` 配下．
- ルートの `Cargo.toml` は `[workspace] members = ["simulation"]`．
- ルートの `pyproject.toml` は uv workspace ルート（`[tool.uv.workspace] members = ["tools"]`）．

### Rust (simulation/src/)

- **main.rs** — CLI エントリポイント．clap の `run` / `sweep` サブコマンド分岐．論文固有のシミュレーションロジックを実装する．

### Python (tools/src/{{NAME}}_tools/)

- **cli.py** — `{{NAME}}-tools` コマンドの統合エントリポイント．argparse のサブパーサで各サブコマンドにディスパッチ．
- **visualize.py** — 単一実行の可視化．`results/latest/` 配下を読み込んで図を生成．

### 出力構造

各実行はタイムスタンプ付きサブディレクトリに保存される．`results/latest` は最新の実行へのシンボリックリンク．

## Key Design Decisions

- 乱数: `ChaCha8Rng` 等の決定的 RNG を使用し，シードによる再現性を確保する想定．
- CLI フラグは clap 規約に従い kebab-case．
- CSV 出力には serde + csv クレートを使用する想定．
- Python CLI は argparse のサブパーサ方式（ネストせず手動ディスパッチ）．
